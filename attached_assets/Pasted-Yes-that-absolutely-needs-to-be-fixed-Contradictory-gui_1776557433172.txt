Yes, that absolutely needs to be fixed. Contradictory guidance kills trust fast.

If the app says:

* Your plan: full rest day
* Symptom action: take a 10-minute walk

the user immediately feels:

“this app is not actually thinking”

That’s a core product integrity issue.

What should be true

Your symptom actions should respect the day’s plan context.

Think of it like this:

* Your Plan = the day’s overall operating mode
* Symptom recommendation = the best action within that mode

So if the plan says:

* full rest day

then symptom actions should adapt to:

* “stand and stretch for 2 minutes”
* “walk lightly around your home”
* “gentle movement only if tolerated”

Not:

* “walk 10 minutes now”

Clean logic

You need a simple hierarchy:

1. Plan context is the guardrail

Possible plan states:

* full rest
* light activity
* normal activity

2. Symptom actions are generated inside that guardrail

Example:

Constipation

* if plan = normal activity → “Walk 10 minutes now”
* if plan = light activity → “Walk 5 minutes gently”
* if plan = full rest → “Stand and move gently for 2 minutes if tolerated”

Low energy

* if plan = full rest → “Stay off your feet and eat something small now”
* not “push through with movement”

Nausea

Usually less affected by movement plan, but still should stay coherent with recovery framing.
