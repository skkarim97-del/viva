Refactor the “Your Day” selection system so each category is a single intentional choice, and make the available options adapt based on the user’s daily state.

The current UX is too messy and inconsistent. Right now each category mixes together actions, habits, tips, and rules. It feels like a random list instead of a clean daily plan.

I want to redesign this so that each wellness category represents one clear decision.

The goal is that the user is choosing one primary action per category for the day.

Examples:
- Mind: choose one thing like 10 min meditation, 15 min hot bath, 10 min journaling
- Move: choose one thing like 45 min cardio, 45 min strength training, 15 min mobility
- Fuel: choose one nutrition approach for the day
- Hydrate: choose one hydration target or hydration plan
- Same logic for any other daily wellness category

This should feel like a coach helping the user commit to a daily plan, not a content dump.

CORE PRODUCT RULE

For every category in “Your Day”:
- The user can select only one primary option
- That selected option becomes the official plan for that category for the day
- The category should not show a bunch of overlapping suggestions that represent different kinds of things
- All options within a category should be at the same level of abstraction and should be mutually exclusive enough to make sense as a single choice

Do not mix:
- actions
- informational tips
- behavior reminders
- broad philosophies
- restrictions

inside the same choice list

Example of bad structure:
- “8+ cups water throughout the day”
- “reduce caffeine after 2 pm”
- “track water intake today”
- “herbal tea in the evening”

These are not parallel choices. Some are goals, some are reminders, some are habits.

Instead, the selectable options should all be the same type of thing.

NEW UX MODEL

Each category should have:
1. One primary selection area
   This is where the user picks one option only.

2. Optional supporting guidance
   After a user selects an option, show 1 to 2 small supporting suggestions below if helpful, but these should not be selectable as separate primary options.

Example:
- User selects: “45 min strength training”
- Supporting guidance below:
  - “Optional: 5 min warmup”
  - “Optional: post-workout stretch”

These are support details, not competing choices.

NEW DAILY STATE LOGIC

I also want each category to adapt based on the user’s daily state:
- Great
- Good
- Tired
- Stressed

This logic should apply across the daily wellness categories so the plan feels personalized and realistic.

Important:
- Each category should still end with one final selected action
- But the options shown and the default recommendation should be influenced by the user’s current state
- The user can always override the recommendation

This means the app should not show the exact same intensity or tone of options every day.

Examples:
- Great: more ambitious options
- Good: solid productive baseline options
- Tired: reduced intensity, recovery-supportive options
- Stressed: calming, restorative, nervous-system-friendly options

For example in MOVE:
- Great: 60 min strength training, 45 min cardio
- Good: 45 min strength training, 30 min cardio, moderate movement
- Tired: 20 min walk, 20 min mobility, recovery movement
- Stressed: gentle walk, yoga, mobility, low-stimulation movement

For example in MIND:
- Great: journaling, meditation, focused reset
- Good: guided meditation, breathing, walk outside
- Tired: hot bath, body scan, short meditation
- Stressed: box breathing, hot bath, calming walk, nervous system reset

For example in FUEL:
- Great: high protein focus, balanced performance meals
- Good: balanced meals, whole-food focus
- Tired: easy digestion, steady blood sugar support
- Stressed: simple nourishing meals, hydration support, lower-friction nutrition

For example in HYDRATE:
- Great: baseline hydration or workout hydration
- Good: baseline hydration with consistency
- Tired: hydration plus electrolytes
- Stressed: hydration plus calming support and reduced overstimulation

The state-based logic should feel intuitive and coach-like, not random.

IMPORTANT IMPLEMENTATION RULE FOR STATE-BASED OPTIONS

Each category should show 4 options total, one aligned to each readiness state:
- Great
- Good
- Tired
- Stressed

That means inside each category selector, the user should see 4 clear options, with each option intentionally mapped to one of those states.

Example structure for MOVE:
- Great → 60 min strength training
- Good → 45 min cardio
- Tired → 20 min recovery walk
- Stressed → 15 min yoga or mobility

Example structure for MIND:
- Great → 10 min journaling
- Good → 10 min guided meditation
- Tired → 15 min hot bath
- Stressed → 5 min box breathing

This creates a simple and elegant system:
- one option per state
- four options per category
- only one final choice selected by the user

The app should visually indicate which option best matches the user’s current state, for example:
- “Recommended for your current state”
- “Best match for today”

But the user should still be able to pick any of the 4 options.

SELECTION BEHAVIOR

Implement the following:

Single select only
Each category must behave like radio buttons, not checkboxes.

The user can select only one option in each category.

If they select a new option, it replaces the old one.

Selected state
The selected option should be very visually obvious:
- highlighted background
- border glow or accent
- checkmark icon
- slightly stronger text contrast

Unselected state
Unselected options should remain tappable but visually secondary.

Tap behavior
When the user taps a category card from “Your Day”, the bottom sheet or modal opens.

Inside the sheet:
- user sees only 4 coherent options for that category
- each option maps to Great, Good, Tired, or Stressed
- one option is visually tagged as the recommended match for the user’s current state
- user taps one option
- selection saves immediately or with a clear confirm CTA depending on current product pattern
- sheet closes or gives brief confirmation feedback

Current card summary
After selection, the main “Your Day” card should show only the selected result.

Example:
- MOVE → 60 min strength training
- MIND → 10 min guided meditation
- HYDRATE → 10+ cups + electrolytes
- FUEL → High protein focus

Not a sentence fragment that sounds inconsistent with the category.

- the wording must match exactly
- no paraphrasing
- no separate local state that drifts out of sync

There should be one source of truth for the day’s selected items.
