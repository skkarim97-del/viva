Keep:

Change “Try This Today” into individual interactive rows.

Example structure:

Personalized check-in
Based on today’s symptoms

WHAT VIVA NOTICED
Viva noticed low appetite, nausea, low energy and constipation in your check-in today.

TRY THIS TODAY

[Row 1]

[Row 2]

Buttons:

[Row 3]

[Row 4]

Feedback buttons:

Bottom:

Design requirements:
- Do NOT show the recommendations as one long paragraph
- Each support item should be visually separated as its own row/cardlet
- Each item should have its own completion buttons: Did this / Didn’t do this
- Keep the copy concise
- Keep the card easy to scan
- Avoid making the full card feel massive or overwhelming
- Use dividers, subtle nested cards or light row backgrounds
- The primary card can still be blue-tinted, but rows should be clean and readable

Behavior:
- Tapping “Did this” for a row should mark that specific recommendation complete
- Tapping “Didn’t do this” should mark that row skipped
- The patient should be able to complete some recommendations and skip others
- Overall feedback can remain one simple Better / Same / Worse row for now
- If “Worse” is selected, show/emphasize “Ask my care team”
- If symptoms change live, update the same card and its rows within ~1-2 seconds
- Do not create multiple full cards

Acceptance:
With low appetite + nausea + tired + constipation selected, the card should show one master Personalized check-in card with 3-4 separate recommendation rows, each with its own Did this / Didn’t do this buttons.
